37signals是一家很特别的公司，他们有很独特的理念。虽然他们的团队只有16人，但他们的几个产品每年都会带来相当可观的利润。他们反对风险投资，反对扩大规模，反对堆砌功能。如果让我用一个词形容他们的话，那就是maverick，在某种程度上他们与乔布斯有相似之处。David Heinemeier Hansson是37signals的一个主要成员，我写过<a href="../David-Heinemeier-Hansson-speech.html">一篇文章</a>介绍他的两个演讲。除了他们的产品外，他们还有一些byproduct: 一个很著名的博客:<a href="http://37signals.com/svn/">Signal vs Noise</a>，还有2本书:《getting real》《rework》，今天我就要介绍一下《rework》这本书。

《getting real》是一本和《rework》风格类似的书，毕竟是同一家公司出的，也值得一读，用不了多少时间。让人兴奋的是《getting real》可以在线免费读，地址是：<a href="http://gettingreal.37signals.com/">http://gettingreal.37signals.com/</a>，而《rework》是要花美金买的，我有电子版的，下载地址在<a href="http://www.box.net/shared/vf4nd4baxs">这里</a>

用一个词形容《rework》这本书就是“短小精悍”，只有130页，你只需要3-4个小时就可以读完，而且书中每一个观点都配了一张图，看起来很轻松，一点都不乏味。我们说“字如其人”，同样的道理，这本书很鲜明反映了37signals这家公司的特点。废话不多说了，我把每个观点都通用一句汉语总结出来了：

<h2>轻装上阵</h2>
<ul>
	<li>不要被教条所束缚</li>
	<li>失败并非成功之母，不要“崇拜”失败，从成功中你可以学到更多</li>
	<li>计划等同于猜测，未来有太多你不能够控制的因素，长远的计划没有任何意义</li>
	<li>企业不应盲目扩张，规模不是衡量企业成功与否的因素，小企业有大企业所没有的优势:灵敏</li>
	<li>不要成为工作狂，你需要时间思考更重要的东西</li>
	<li>“创业家”(entrepreneur)这个词应该被替换成“起步者”(startup)，每个人都可以开创自己的事业</li>
</ul>
<h2>出发</h2>
<ul>
	<li>做一些能够引起轰动的事，最关键的是能够真正帮助别人解决一些问题，而不是去山寨别人的产品</li>
	<li>抓住自己的需求，设计能够解决自己所遇到的问题的产品</li>
	<li>立即开始去行动，你的计划和想法并不代表你做了什么</li>
	<li>“没有时间”是借口，做你真正喜欢的事你不会说“没有时间”</li>
	<li>必须有自己的信条，它是设计产品时的准绳</li>
	<li>努力实现你的承诺，捍卫你的信条，而不是把它作为口号挂在墙上</li>
	<li>拿VC的钱是最差的选择，花别人的钱你会上瘾，最终失去对于公司的控制</li>
	<li>你所需要的并不多，创业公司不是大公司，你需要的只是一个车库罢了</li>
	<li>不要用“创业公司(startup)”这个词麻痹自己了，你需要的是盈利，你需要开始真正的事业</li>
	<li>你应该考虑如何让自己的企业获得成功，而不是尽早拿着现金退出</li>
	<li>抛弃那些没有意义的东西吧,比如：长远的计划，过量的员工，无聊的会议，繁琐的制度</li>
</ul>
<h2>前进</h2>
<ul>
	<li>“少”是一件好事，限制更是一种优势</li>
	<li>一段时间只做一件事，做最重要的事，把全部精力投入到一个产品上去</li>
	<li>从最核心的地方开始，细枝末节的部分以后慢慢修改</li>
	<li>开始的时候不要太关注细节，它只会浪费你的时间</li>
	<li>不断做决定的过程就是不断前进的过程，不要优柔寡断</li>
	<li>抓住最核心的东西，抛弃那些不重要的东西，真正重要的东西迟早会引起你的注意</li>
	<li>少即是多，不做最多的，而做最对的</li>
	<li>专注于那些不会改变的东西，而不是那些时髦的新事物</li>
	<li>问题的关键是争取顾客并赚钱，而不是你使用什么工具。一跟贵的高尔夫球杆不能保证你打球出色</li>
	<li>买出你的“副产品”，做任何事总会给你带来“副产品”，他们可能同样具有价值</li>
	<li>不要总被细枝末节所羁绊，不要等到尽善尽美才发布，尽快发布产品，并且发布后不断改进</li>
</ul>
<h2>效率</h2>
<ul>
	<li>现实一点吧，用最直观的方式描述事物，放弃那些没有任何意义的文档和图表</li>
	<li>不要只顾低头干事，还要抬起头来问问自己究竟为什么要这样做，这件事真的有意义吗</li>
	<li>中断是高效的天敌，拿出一块完整的连续的时间做一件事，这样的效率最高</li>
	<li>开会就像毒药</li>
	<li>足够好就可以了，不必苛求完美，不必最求繁杂，那只会满足你的虚荣心</li>
	<li>速度第一，尽快实现一个一个小的目标，这会激励你实现最终的大目标</li>
	<li>放弃不等于失败，该放弃时就放弃，逞能只会让你失去的更多</li>
	<li>注意休息，缺乏睡眠会让你失去创造力，士气低落，并且变得暴躁易怒</li>
	<li>把大计划分成小计划，把大块时间切割成小块时间，这会帮助你更好的做出计划和选择</li>
	<li>任务清单要小，每次只关心一件事</li>
	<li>不断做出小的决定，并且不断根据时机而改变</li>
</ul>
<h2>对手</h2>
<ul>
	<li>不要抄袭，通过抄袭你无法理解事物的本质，而且你一直处于被动的地位</li>
	<li>把你自身融入到产品中去，变成产品的一部分，这是对手所抄袭不来的</li>
	<li>找出对立的对手，让自己变得鲜明，同时也利于产品的营销</li>
	<li>不要通过冷战的方式与对手较量，你需要的不是比对手做的更多，而是更少更精</li>
	<li>把精力集中于自己，而不是竞争对手</li>
</ul>
<h2>进化</h2>
<ul>
	<li>学会说不，不能被顾客牵着鼻子走，要有自己的原则</li>
	<li>不要专门为一两个大客户而改变产品，你需要专注于一类用户，而不是一两个</li>
	<li>想到好的想法不要头脑发热，先把它写下来，日后仔细考虑，互相比较，再决定实现哪一个</li>
	<li>做那种让用户越用越喜欢的东西，而不是买完就后悔的东西，这样他会向他的朋友推荐你的产品，这是最好的广告</li>
	<li>没有必要把用户反馈都记录下来，真正重要的建议你会多次接触，自然能记住</li>
</ul>
<h2>晋级</h2>
<ul>
	<li>开始的时候默默无闻是一件好事，它给你犯错误的空间</li>
	<li>培养你自己的听众，让他们主动了解你，博客是一个很不错的方式</li>
	<li>“教育”用户，让他们对你“忠诚”，他们会尊敬你，apple是个好例子</li>
	<li>学会分享，分享不会使你被别人超越，人们会喜欢你，你将从中收益</li>
	<li>向用户展示产品背后的故事，这会拉近你与用户的关系</li>
	<li>做真实的自己，正视自己的不足，残缺也是一种美</li>
	<li>不要向记者们群发新闻稿，约他们出来，富有激情的向他们推销，这样你会获得最好的报道</li>
	<li>不要指望通过那些大型媒体营销自己，相反那些草根博主也许可以帮助你</li>
	<li>开始时免费给用户提供一些服务，如果他们喜欢，他们会花钱使用你更完善的服务</li>
	<li>没有必要专门设置“销售部”，你发的每封邮件，回的每个电话都是在marketing</li>
	<li>一夜成名的背后必然是数年的心血和默默的付出</li>
</ul>
<h2>招聘</h2>
<ul>
	<li>招聘之前自己先体验一下这个岗位，这不仅有助于找到合适的人，还会帮助你日后更好的管理</li>
	<li>在真正需要时才招聘，而不是仅仅为了扩大规模</li>
	<li>不要因为对方很优秀就把他招进来，招最合适的，而不是最好的</li>
	<li>不要不停大量招新人，这会使每一位员工身边都有大量陌生人，这对于公司发展很不利</li>
	<li>同一份简历同时发给很多公司的人不要招。如何辨别这样的人：看信中他是在与你沟通，还是在堆砌要点</li>
	<li>5年工作经验与半年工作经验没有太大区别，你很难辨别其间的区别</li>
	<li>GPA并不重要，除非你需要一个教授，但事实上你不需要</li>
	<li>对于小团队，每个成员都必须实实在在地做事，你不需要只会指手画脚的人</li>
	<li>你需要一个能够从小事做起，并且总揽大局的管理者</li>
	<li>聘用有更好写作能力的人，他们有更清晰的思维，更好的表达能力</li>
	<li>合适的人选在世界各地，没必要在乎地理位置，是否在同一个城市对于一个团队并不重要</li>
	<li>可以给应聘者一个小项目做，这是检验他能否胜任的最好方式</li>
</ul>
<h2>应对问题</h2>
<ul>
	<li>当负面新闻出现时，坦诚地向消费者解释，而不是试图掩盖它</li>
	<li>迅速解决客户遇到的问题，对方不但不会抱怨，反而会更加信任你</li>
	<li>对于道歉要学会换位思考：如果你是被道歉者，你想听到怎样的道歉？最重要的是真诚</li>
	<li>让每一个员工都能接触到客户，对于任何岗位这都有好处</li>
	<li>发布新版本后深吸一口气吧，抵制改变是人的天性。不要消极应对客户的抱怨，而是要耐心的向他们解释</li>
</ul>
<h2>企业文化</h2>
<ul>
	<li>企业文化不是你能够创造的，它是在长时间内一点点形成的</li>
	<li>决定都是暂时的，要随着形势的不断变化而不断改变决定</li>
	<li>企业的文化必须尊重每一个人的工作</li>
	<li>不要像对待孩子一样对待员工，你应该信任他们，给予他们足够的空间</li>
	<li>不要强行加班，这会降低员工白天的工作效率，最终不会给企业带来任何好处</li>
	<li>不要往别人的伤口上撒盐，也不要惩罚那些偶然出现的问题，你应该关注那些经常出现的问题</li>
	<li>不要装腔作势，那其实是一种自卑的表现</li>
	<li>不要出现need, must, can't, easy, just, only, fast这几个单词</li>
	<li>只将真正紧急的事情贴上紧急标签</li>
</ul>
<h2>结语</h2>
<ul>
	<li>当灵感出现时，立即抓住它，不要拖延。你可以有很多idea，但如果不实践，他们都等于零</li>
</ul>
